───────────────────────────────────────────────────────────────────────────────
  VALIDATION RULES TO TEST
───────────────────────────────────────────────────────────────────────────────

1. FILE FORMAT VALIDATION
   ├─ LC-MS/MS (ID=1) accepts: MZML, CDF
   │  ├─ ✓ TEST_DATA_LCMSMS_CHROMATOGRAM.csv (CSV contains mzML data)
   │  └─ ✓ TEST_DATA_LCMSMS_MZML_SIMULATION.csv
   ├─ HPLC (ID=2) accepts: CSV, PDF
   │  ├─ ✓ TEST_DATA_HPLC_ANALYSIS.csv
   │  └─ ✓ TEST_DATA_HPLC_REPORT.pdf
   └─ Dissolution (ID=3) accepts: CSV
      └─ ✓ TEST_DATA_DISSOLUTION_APPARATUS.csv

2. CALIBRATION DATA VALIDATION (R²)
   ├─ Acceptance: R² ≥ 0.99
   ├─ LC-MS/MS file: R² = 0.9997 → PASS ✓
   ├─ HPLC file: R² = 0.9999 → PASS ✓
   └─ To test failure, you would modify CSV: R² = 0.98 → FAIL ✗

3. QC PASS RATE VALIDATION
   ├─ Acceptance: ≥80% pass rate
   ├─ All test files: 100% pass rate → PASS ✓
   └─ To test failure, modify CSV: Set 3/9 QC to FAIL → 67% → FAIL ✗

4. PEAK SYMMETRY & RETENTION TIME
   ├─ Tailing Factor: 0.80-1.20
   ├─ All test files: 0.95-0.98 → PASS ✓
   └─ System should warn if outside range

───────────────────────────────────────────────────────────────────────────────
  TROUBLESHOOTING
───────────────────────────────────────────────────────────────────────────────

PROBLEM: "File format not accepted"
└─ Cause: File extension doesn't match instrument
└─ Solution:
   ├─ Check Tab 1: What instrument was selected?
   ├─ LC-MS/MS → Use: TEST_DATA_LCMSMS_*.csv files
   ├─ HPLC → Use: TEST_DATA_HPLC_*.csv or .pdf
   └─ Dissolution → Use: TEST_DATA_DISSOLUTION_*.csv

PROBLEM: "Failed to parse file"
└─ Cause: File format doesn't match expected structure
└─ Solution:
   ├─ Try different test file
   ├─ Check if backend has parsing logic for that format
   ├─ Verify file is not corrupted
   └─ Check console logs for detailed error

PROBLEM: "Calibration R² below 0.99"
└─ Cause: File has low R² value
└─ Solution:
   ├─ All test files have R² ≥ 0.9997
   ├─ If error occurs, file may be modified
   ├─ Restore from original
   └─ Contact QA for method validation

PROBLEM: "QC pass rate < 80%"
└─ Cause: File has QC failures
└─ Solution:
   ├─ All test files have 100% pass rate
   ├─ If error occurs, file may be modified
   ├─ Restore from original
   └─ To intentionally test failure, modify CSV

PROBLEM: No data extracted after upload
└─ Cause: Backend parsing not working
└─ Solution:
   ├─ Check if analyzer-data/upload endpoint exists
   ├─ Verify endpoint is called (check browser network tab)
   ├─ Check backend logs for parsing errors
   ├─ Verify file is received by backend
   └─ May need to enhance parsing logic
