# Stage 7: Data Handling & Analysis

**Branch**: `feat/053-tb-lab-workflow` | **Date**: 2025-12-24 **Parent Spec**:
`specs/053-tb-lab-workflow/spec.md`

## Overview

Stage 7 handles the final phase of the TB workflow: result compilation,
reporting, data export, and integration with external systems (REDCap). This
stage also includes a summary dashboard for workflow metrics and status
tracking.

**Key Workflow:** All Tests Complete → Result Compilation → Report Finalization
→ Export (CSV/JSON/REDCap) → Dashboard Tracking

## User Stories

### US7.1 - Result Compilation (Priority: P2)

Lab staff compile all test results for a sample into a unified report.

**Acceptance Scenarios:**

1. **Given** sample with complete testing, **When** viewing compilation page,
   **Then** all results displayed: Registration, QC, Culture, Smear, GeneXpert,
   DST, Storage
2. **Given** compilation ready, **When** finalizing, **Then** "Reported by",
   "Reviewed by", comments captured
3. **Given** report finalized, **Then** printable/exportable report generated
4. **Given** incomplete testing, **When** viewing, **Then** missing tests
   indicated with warning

### US7.2 - Report Generation (Priority: P2)

Generate printable TB result reports following standard format.

**Acceptance Scenarios:**

1. **Given** finalized compilation, **When** generating report, **Then**
   PDF/print-ready format created
2. **Given** MDR/XDR flags, **When** generating report, **Then** resistance
   pattern prominently displayed
3. **Given** QC warnings, **When** generating report, **Then** QC issues noted
   in report
4. **Given** report generated, **Then** generation date and generated by
   recorded

### US7.3 - Data Export (Priority: P3)

Export sample data in various formats for external systems.

**Acceptance Scenarios:**

1. **Given** samples selected for export, **When** exporting, **Then**
   CSV/JSON/Excel format available
2. **Given** REDCap export, **Then** column mapping matches REDCap data
   dictionary
3. **Given** export complete, **Then** `exported_at` timestamp recorded per
   sample
4. **Given** date range filter, **Then** only samples in range included

### US7.4 - Summary Dashboard (Priority: P3)

View workflow metrics and sample status summary.

**Acceptance Scenarios:**

1. **Given** TB workflow active, **When** viewing dashboard, **Then** sample
   counts by stage displayed
2. **Given** filters applied (date, status), **Then** metrics update accordingly
3. **Given** time period selected, **Then** workflow throughput chart displayed
4. **Given** alerts present (MDR/excursions), **Then** alert summary shown

## Database Schema

### tb_result_report

```sql
CREATE TABLE tb_result_report (
    id SERIAL PRIMARY KEY,
    notebook_page_sample_id INTEGER NOT NULL REFERENCES notebook_page_sample(id),

    compilation_date DATE NOT NULL,
    reported_by VARCHAR(100),
    reviewed_by VARCHAR(100),
    comments TEXT,

    status VARCHAR(20) NOT NULL DEFAULT 'DRAFT' CHECK (status IN ('DRAFT', 'FINALIZED')),
    finalized_date DATE,

    mdr_flag BOOLEAN DEFAULT FALSE,
    xdr_flag BOOLEAN DEFAULT FALSE,
    qc_warning BOOLEAN DEFAULT FALSE,

    report_file_path VARCHAR(500),
    generated_date TIMESTAMP,
    generated_by INTEGER REFERENCES system_user(id),

    lastupdated TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT uq_report_sample UNIQUE (notebook_page_sample_id)
);

CREATE INDEX idx_result_report_nps ON tb_result_report(notebook_page_sample_id);
CREATE INDEX idx_result_report_status ON tb_result_report(status);
CREATE INDEX idx_result_report_mdr ON tb_result_report(mdr_flag);
```

### tb_export_log

```sql
CREATE TABLE tb_export_log (
    id SERIAL PRIMARY KEY,
    notebook_entry_id INTEGER REFERENCES notebook_entry(id),

    export_date TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    format VARCHAR(20) NOT NULL CHECK (format IN ('CSV', 'JSON', 'EXCEL', 'REDCAP')),
    sample_count INTEGER NOT NULL,
    export_file_path VARCHAR(500),

    redcap_success BOOLEAN,
    redcap_project_id VARCHAR(100),
    redcap_response TEXT,

    exported_by INTEGER REFERENCES system_user(id),
    lastupdated TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_export_log_date ON tb_export_log(export_date);
CREATE INDEX idx_export_log_format ON tb_export_log(format);
```

### tb_sample_export_status

```sql
CREATE TABLE tb_sample_export_status (
    id SERIAL PRIMARY KEY,
    notebook_page_sample_id INTEGER NOT NULL REFERENCES notebook_page_sample(id),
    export_log_id INTEGER REFERENCES tb_export_log(id),
    exported_at TIMESTAMP NOT NULL,
    exported_to VARCHAR(50) NOT NULL,

    CONSTRAINT uq_sample_export_dest UNIQUE (notebook_page_sample_id, exported_to)
);

CREATE INDEX idx_sample_export_nps ON tb_sample_export_status(notebook_page_sample_id);
```
